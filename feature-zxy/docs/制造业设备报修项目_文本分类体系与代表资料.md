# 制造业设备报修文本故障分类与停机风险预警：文本分类体系与代表资料

## 1. 数据建设原则

本项目最核心的问题是数据。公开互联网中很难直接获得十几万条真实中文制造业设备报修工单，因此建议采用“分层数据融合”的方式：

- 用真实公开维护工单数据学习工单文本风格、字段结构、工单状态、优先级。
- 用工业故障诊断和预测性维护数据提供真实故障事实、失效标签、停机风险标签。
- 用设备手册、报警码手册、维修案例补充专业术语。
- 用受控改写扩展自然语言表达，但不让大模型凭空生成故障标签。
- 用人工抽检样本作为测试集，保证项目答辩时可信。

## 3. 可落地的模型任务

### 3.1 故障大类分类

输入报修文本，输出机械、电气、传感器、控制系统、工艺参数、安全风险、预防性维护等类别。

### 3.2 停机风险等级分类

输出 P0/P1/P2/P3：

- P0：已停机、无法启动、存在安全风险、影响主线。
- P1：短期可能停机，异常明显，影响产能或质量。
- P2：局部异常，暂未停机，但影响效率。
- P3：普通保养、咨询、备件申请。

### 3.3 处理部门推荐

输出机械维修、电气维修、自动化工程师、仪表工程师、安全/EHS、设备保养等。

### 3.4 故障部位识别

识别主轴、电机、变频器、PLC、传感器、液压站、气缸、皮带、轴承、刀具、冷却水泵等实体。

### 3.5 相似历史工单召回

用 BERT/Sentence-BERT 向量检索相似历史工单，返回历史原因、维修动作、备件消耗和停机时长。

## 4. 代表性公开数据源与适配方式

### 4.1 直接可用于文本训练的数据

1. Industrial Maintenance Synthetic Dataset  
   这是最适合作为训练主语料的数据源。它提供 100 万级维护工单，覆盖 37 类工业设备，包含维护记录文本、工单类型和设备类型。虽然是合成数据，但它的字段和工业设备覆盖度很适合做 BERT/FastText 文本分类训练。

2. MaintNet  
   MaintNet 面向预测性维护语言资源，覆盖航空、汽车、设施维护等领域的维护记录。它的价值在于技术文本中存在大量缩写、拼写不规范和领域术语，和制造业报修文本非常相似。

3. NYC AMPS Work Orders  
   这是纽约市资产管理工单，包含真实工单描述、工单类型、优先级、对象、任务类型等字段。虽然不是制造业，但是真实维护工单，对训练“工单文本理解”和“优先级分类”有迁移价值。

4. NYC Street Sign Work Orders  
   这是大规模真实工单数据，接口查询约 1500 万级记录。它有 order_type、record_type、sign_description、sign_location 等字段，适合练习大规模工单清洗、分类和字段映射。

5. Austin Street Sign Work Orders  
   已查询约 5.4 万条记录。它的 Work Order Status、Work Area、Requester 等字段可用于工单状态分类、工作区域分类和请求来源分类。

6. Louisville PM Workorders  
   字段中有 WORK_DESCRIPTION，可作为真实维护描述样本。适合补充自然语言报修表达。

### 4.2 可转成报修文本的工业故障数据

1. UCI AI4I 2020 Predictive Maintenance Dataset  
   10,000 条结构化预测性维护数据，可将故障类型转成报修文本：TWF 对应刀具磨损，HDF 对应散热异常，PWF 对应功率异常，OSF 对应过载，RNF 对应随机故障。

2. NASA PCoE 数据仓库  
   包含轴承、铣削、电池、航空发动机、IGBT、MOSFET、CNC 铣削等退化数据。适合生成“轴承退化”“刀具磨损”“功率器件过热”“发动机退化”“CNC 刀具寿命不足”等文本。

3. PHM Society Data Repository  
   提供大量 PHM 竞赛数据，是扩展工业故障数据的重要入口。

4. MetroPT  
   来自地铁空气压缩机系统，1.6 GB，含压力、温度、电流和控制信号。适合生成空压机压力异常、温度异常、电流异常、保护停机类文本。

5. SCANIA Component X  
   真实公司数据，约 1.54 GiB，包含 operational data、repair records、specifications。适合构造发动机部件维护记录、时间到事件和异常检测文本。

6. APS Failure at Scania Trucks  
   训练集 60,000 条，测试集 16,000 条，共 76,000 条，适合构造空气压力系统故障、气动异常、压力不足和漏气类文本。

7. MIMII Dataset  
   覆盖风扇、泵、阀门、滑轨等设备异常声音。可转成“风机异响”“泵运行声音异常”“阀门动作异常”“滑轨磨损”等巡检文本。

8. CWRU Bearing Data Center  
   轴承正常和故障振动数据，包含内圈、外圈、滚动体故障。适合机械故障细分类。

9. XJTU-SY Bearing Datasets  
   15 个滚动轴承全寿命退化数据，适合构造停机风险、剩余寿命、机械退化类文本。

10. UCI Hydraulic Systems  
   可用于液压泵、阀、冷却效率、蓄能器、油液状态异常等文本构造。

## 5. 数据量达到十几万的可执行组合

建议目标：构建 15 万到 25 万条训练样本。

| 来源 | 目标样本量 | 作用 |
|---|---:|---|
| Industrial Maintenance Synthetic Dataset | 100,000 到 150,000 | 主训练集，覆盖工业维护文本 |
| 真实公开工单数据 | 20,000 到 50,000 | 学习真实工单表达、状态、优先级 |
| AI4I / NASA / PHM / MetroPT / Scania 转文本 | 30,000 到 80,000 | 构造真实故障事实和风险标签 |
| 设备手册/报警码/维修案例 | 10,000 到 30,000 | 补充专业术语和报警码 |
| LLM 受控改写 | 30,000 到 80,000 | 扩充表达方式，不改变标签 |

训练集可以使用 15 万到 25 万条；验证集和测试集建议单独抽 1 万到 2 万条，并进行人工抽检。

## 6. 数据真实性控制方法

### 6.1 标签不由大模型决定

大模型只负责改写句子，不负责生成故障事实和标签。标签必须来自：

- 原始数据字段。
- 设备故障规则。
- 报警码手册。
- 人工审核。

### 6.2 结构化事实转文本

示例：

```text
设备类型：空压机
异常字段：温度过高
报警码：TEMP_HIGH
状态：自动停机
风险等级：P0
```

可转成：

```text
空压机温度持续升高，现场报警 TEMP_HIGH，随后自动停机。
3 号空压机高温报警，设备已经保护停机，怀疑冷却系统异常。
空压机运行一段时间后温度偏高，影响产线供气，需要立即处理。
```

### 6.3 人工抽检比例

建议：

- 训练集抽检 2% 到 5%。
- 验证集抽检 10%。
- 测试集尽量 100% 人工确认。

### 6.4 去重和泄漏控制

- 相同模板改写样本不能同时出现在训练集和测试集。
- 同一设备、同一报警码、同一时间段生成的文本要按组划分。
- 测试集优先使用真实公开工单和人工编写样本。

## 7. 代表性研究资料内容摘要

1. NIST：Discovering Critical KPI Factors from Natural Language in Maintenance Work Orders  
   该研究说明维护工单自然语言中包含设备、动作、环境和 KPI 相关因素，可用于发现维护策略中的痛点、异常模式和关键资产。它支撑本项目从工单文本中识别故障部位、风险等级和停机影响。

2. Technical Language Processing for PHM  
   该研究指出维护工单文本包含重要系统和部件信息，但经常被当作“黑盒”闲置。论文使用 TF-IDF、BERT、主题建模等方法处理维护工单，和本项目中的 BERT 分类、相似工单召回高度相关。

3. Automated analysis and assignment of maintenance work orders using NLP  
   该研究把维护工单自动分派建模为 NLP 分类任务，说明工单自动分派可以减少人工处理成本。本项目中的“推荐处理部门”可以参考这一思路。

4. Combining NLP and multidimensional classifiers to predict and correct CMMS metadata  
   该研究针对 CMMS 失败通知文本预测和纠正故障代码，支持本项目采用多任务分类或多标签分类，而不是简单单标签分类。

5. Natural Language Processing Model for Managing Maintenance Requests in Buildings  
   该研究说明设施维护请求可以通过 NLP 分类，虽然场景是建筑设施，但方法可以迁移到制造业设备运维。

6. Text-mining building maintenance work orders for component fault frequency  
   该研究说明工单描述可以用于统计部件故障频率，支撑本项目的“故障部位识别”和“高频故障看板”。

7. Generating Semantic Matches Between Maintenance Work Orders for Diagnostic Decision Support  
   该研究使用 BERT 类模型做维护工单语义匹配，可作为本项目“相似历史工单召回”的依据。

8. Adapting NLP for technical text  
   该资料强调技术文本含大量缩写、拼写错误、领域术语，普通 NLP 管道直接处理效果有限。本项目需要工业词表、报警码词典和领域化预处理。

## 8. 项目中可以使用的标签字段设计

建议最终数据表字段如下：

| 字段 | 说明 |
|---|---|
| text | 报修文本 |
| equipment_type | 设备类型 |
| fault_category | 故障大类 |
| fault_part | 故障部位 |
| symptom | 故障现象 |
| risk_level | 停机风险等级 |
| department | 推荐处理部门 |
| alarm_code | 报警码 |
| downtime_flag | 是否已停机 |
| source_type | 数据来源 |
| is_synthetic | 是否为合成/改写样本 |
| confidence_rule | 标签来源置信度 |
